# 프로젝트 작업 가이드 (CLAUDE)

## 🔄 멀티 기기 작업을 위한 동기화 점검 규칙
사용자는 여러 기기(Mac, Windows 등)에서 이 프로젝트를 작업하므로, 작업 시작 전에 반드시 원격 상태를 확인한다.

- **첫 편집/수정/기능 요청을 받으면**, 실제 작업을 시작하기 전에 아래 순서로 동기화 상태를 점검한다.
  1. `git fetch origin`
  2. `git status -sb` 또는 `git log HEAD..origin/master --oneline` 으로 로컬과 원격 비교
  3. 원격이 앞서 있으면(behind), "**원격에 N개의 커밋이 있습니다. 먼저 `git pull` 을 실행할까요?**" 라고 사용자에게 알리고 작업을 멈춘다.
  4. uncommitted 로컬 변경이 있으면 그 내용도 함께 요약해서 보고한다.
  5. 로컬이 원격보다 앞서 있거나 동일하면 그대로 작업을 진행한다.
- 단순 조회성 질문(코드 읽기, 설명 등)은 이 점검을 생략해도 된다.
- 사용자가 명시적으로 "pull 하지 말고 수정만 해줘" 라고 지시하면 점검을 생략한다.

## 🛠 CLI 도구 활용 (gws cli)
- **성공 패턴 (Redirect & Read)**: `gws cli`를 사용할 때 결과를 바로 터미널로 출력하지 않고, 파일로 리다이렉션하여 저장한 뒤 읽는 방식이 가장 빠르고 정확합니다.
  - 예시: `gws sheets +read ... > output.json && cat output.json`
- **장점**: 
  - 쉘(Shell)의 인코딩 문제로 인한 한글 깨짐을 완벽하게 방지합니다.
  - 대량의 데이터를 AI가 파싱하기 가장 좋은 깨끗한 상태로 유지합니다.
  - 여러 번의 `help` 조회나 재시도 없이 단 한 번에 성공할 수 있는 검증된 방식입니다.
